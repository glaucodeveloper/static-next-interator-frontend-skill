Generator nativo · state patch · DOM vivo

O componente é um programa suspenso.

Uma única function* guarda estado em this, cria um DocumentFragment, retorna um HTMLElement vivo e recebe a próxima transição por next(patch).

Tutorial principal

Anatomia completa, parte por parte

O contador é o exemplo máximo mínimo: contém tudo que um componente stateful precisa. Interfaces maiores apenas ampliam o shape do estado, os métodos e o template.

handler→this.next(patch)→state→fragment→replaceWith→yield root
1

Binder mínimo

Cria um contexto independente, chama o generator com esse contexto como this e liga os métodos nativos do iterator.

2

Estado e métodos em this

id, state, element e handlers pertencem à instância, não a registries globais.

3

Template inline vira fragmento

O template descartável nasce dentro de Object.assign(document.createElement("template"), { innerHTML }). Seu .content é consumido imediatamente, sem variável local.

4

Root estendido

element.component = this liga o nó atual ao contexto vivo que contém métodos e estado.

5

Commit localizado

Se o root anterior está conectado, replaceWith() o troca. A referência this.element passa a apontar para o novo root.

6

Yield bidirecional

Entrega o elemento atual e recebe o patch enviado pela próxima chamada a next(patch).

function component(Component, props) {
  const context = {};
  const iterator = Component.call(context, props);
  context.next = iterator.next.bind(iterator);
  context.return = iterator.return.bind(iterator);
  context.throw = iterator.throw.bind(iterator);
  return context;
}

function* CounterComponent({ id, props = {} }) {
  const initialState = {
    counting: props.counting ?? 0,
    label: props.label ?? "Contador",
  };

  this.id = id;
  this.state = { ...initialState };
  this.element = null;

  this.increment = amount => this.next({
    counting: this.state.counting + amount,
  });
  this.reset = () => this.next(initialState);

  while (true) {
    Object.assign(
      this.state,
      yield (this.element = ((element) => {
        element.id = this.id;
        element.component = this;

        if (this.element?.isConnected) {
          this.element.replaceWith(element);
        }

        return element;
      })(Object.assign(document.createElement("template"), {
        innerHTML: `<section>
          <output>${this.state.counting}</output>
          <button onclick="document.getElementById('${id}').component.increment(1)">+1</button>
          <button onclick="document.getElementById('${id}').component.reset()">reset</button>
        </section>`,
      }).content.firstElementChild)),
    );
  }
}

O ponto decisivo

Object.assign(state, yield element)

A expressão completa une a saída da renderização atual à aplicação automática do próximo state patch.

A

Saída

component.next() avança até o yield e recebe { value: element, done: false }.

const first = counter.next();
app.append(first.value);
B

Entrada

component.next(patch) retoma o ponto suspenso. O argumento vira o valor de yield element.

counter.next({ counting: 10 });
// patch === { counting: 10 }

O primeiro next() suspende no yield e entrega o root. O próximo next(patch) retoma a expressão; seu argumento torna-se o segundo argumento de Object.assign, atualiza this.state e libera a próxima volta do render.

Filosofia de escrita

O código é a linha temporal do componente

O template inline não é um truque de concisão. Sua posição dentro do yield preserva a sequência exata: produzir a interface atual, suspender, receber o StatePatch, concluir a atualização e produzir a interface seguinte.

abre assign→cria root inline→yield suspende→next(patch)→assign conclui→novo render
1

Atualização pendente

Object.assign(this.state, ...) abre a operação, mas ainda não possui o segundo argumento.

2

Produção transitória

O template é criado, preenchido e consumido inline. Ele não recebe nome porque não possui identidade nem duração no componente.

3

Fronteira de suspensão

O root é entregue por yield. A chamada externa recebe o HTMLElement; o assign continua incompleto.

4

Recebimento do StatePatch

this.next(statePatch) retoma exatamente o ponto suspenso. O patch vira o valor da expressão yield (...).

5

Conclusão e repetição

O Object.assign finalmente atualiza o estado. O while continua e o próximo root nasce do estado novo.

Genealogia

Inspiração clara em Effect/Effect-TS

A influência está na separação entre descrever um programa suspenso e interpretá-lo explicitamente. Static Next especializa essa ideia para o DOM: a função generator descreve o ciclo, o binder dirige next/throw/return, o valor produzido é um HTMLElement e o valor de retomada é um StatePatch.

É inspiração conceitual, não dependência ou compatibilidade de API. O runtime usa somente JavaScript e DOM nativos.
