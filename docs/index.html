JavaScript puro · estado por instância · rerender localizado

Um componente inteiro, explicado por dentro.

Static Next usa módulos estáticos com this.frontends, this.events, this.template() e this.update(). Sem generators, factories ou ciclos paralelos.

Primeira leitura

Anatomia canônica completa

O contador abaixo é o componente mínimo completo. As interfaces mais complexas repetem exatamente esta estrutura.

onclick→events[id]→this.update→this.template→outerHTML
1. Registries: instâncias e handlers

frontends[id] contém estado e root atual. events[id] contém as ações públicas daquela instância.

const CounterComponent = {
  frontends: {},
  events: {},
};
2. mount(): cria a instância e captura this

As arrows preservam o this do módulo. Todo handler calcula um patch e chama this.update().

mount(id, target = document.body) {
  this.frontends[id] = { state: { counting: 0 }, element: null };
  this.events[id] = {
    add: amount => this.update(id, {
      counting: this.frontends[id].state.counting + amount,
    }),
    reset: () => this.update(id, { counting: 0 }),
  };
  target.insertAdjacentHTML("beforeend", this.template(id));
  this.frontends[id].element = document.querySelector(`#${CSS.escape(id)}`);
}
3. template(): render derivado do estado

O template lê a instância registrada e devolve um único root. O HTML chama o handler pelo registry público.

template(id) {
  const { counting } = this.frontends[id].state;
  const idRef = JSON.stringify(id);
  return `<section id="${id}">
    <output>${counting}</output>
    <button onclick="CounterComponent.events[${idRef}].add(1)">+1</button>
    <button onclick="CounterComponent.events[${idRef}].reset()">reset</button>
  </section>`;
}
4. update(): auto state e rerender

Esta é a única transição: valida, mescla, gera o próximo HTML, substitui o root e atualiza a referência.

update(id, patch = {}) {
  const frontend = this.frontends[id];
  validatePatch(patch);
  Object.assign(frontend.state, patch);
  frontend.element.outerHTML = this.template(id);
  frontend.element = document.querySelector(`#${CSS.escape(id)}`);
  return frontend.element;
}

Ponto crítico

Handlers possuem uma responsabilidade só

Capturar intenção

O template escolhe a ação e seus argumentos.

onclick="Component.events[id].add(1)"

Calcular patch

A arrow lê o estado atual usando this.

add: amount => this.update(id, {
  counting: this.frontends[id].state.counting + amount,
})

Delegar update

O handler não substitui HTML nem mescla estado por conta própria.

this.update(id, patch)
